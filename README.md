# Financial Intelligence Multi-Agent System

A personalised financial research and intelligence platform powered by a multi-agent architecture. The system aggregates data from multiple specialised sources, applies agent reasoning, and delivers personalised insights, reports, and digests tailored to each user's followed entities and risk appetite.

> **Status:** Pre-development — requirements and architecture phase  
> **V1 Scope:** US markets, US-listed companies, US economic indicators, US market indices

---

## What This System Does

This is not a trading platform and does not provide financial advice. It is a research intelligence tool for self-directed investors who want:

- **Aggregated, reasoned financial intelligence** across companies, sectors, indices, and economic indicators they follow
- **Personalised weekly digests** covering market trends, macro context, and followed entities
- **On-demand research reports** generated by agents performing deep retrieval across multiple specialised sources
- **A stateful conversational interface** that knows their history, preferences, and risk appetite
- **Real-time trend and momentum signals** surfaced proactively via a market dashboard

---

## Key Features

### Personalisation
Every output — digests, reports, chat responses, trend signals — is personalised through the user's risk appetite (conservative, moderate, aggressive), followed entities, and interaction history.

### Multi-Agent Architecture
An orchestrator agent decomposes user queries and dispatches them concurrently to specialised sub-agents, each with their own tools and data sources. The orchestrator applies iterative self-validation before returning answers, asking clarifying questions rather than returning low-confidence responses.

### Summarisation Service
A core system-wide capability that maintains structured, up-to-date summaries per tracked entity and per user. Entity summaries are sectioned by data type (news, analyst sentiment, filings, market context) and serve as the primary knowledge input for agent operations. User context summaries are consumed by the orchestrator at the start of every interaction.

### Data Ingestion Pipelines
Multiple independent pipelines operate at cadences appropriate to each data type — daily for market prices, scheduled for news, aligned to release calendars for economic indicators, and daily for SEC filings via EDGAR.

### Report Generation
Professional PDF reports generated on demand, stored in object storage, and delivered via presigned URLs. Reports perform deep retrieval from raw sources rather than relying on summaries, and cite all sources including agent reasoning.

## Project Goals

This project is built with two parallel objectives:

1. **Build something genuinely useful** — a production-grade multi-agent financial intelligence system with real architectural complexity
2. **Learn and practice high-value engineering concepts** — multi-agent orchestration, Graph RAG, event-driven architecture, observability, async programming, API design, infrastructure as code, and evaluation frameworks
